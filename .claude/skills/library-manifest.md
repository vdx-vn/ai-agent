# Library Manifest

| Skill | Class | Category | Primary artifact | Key tie-breaker | Compose with |
|---|---|---|---|---|---|
| `odoo-think` | task | Sprint task skills | Risk-oriented scoping brief with module map, business entrypoint, cross-app impact, and open questions. | Use this skill only when the primary requested output is a scoping brief or impact diagnosis. If the user wants ordered steps, file-by-file actions, or acceptance criteria, hand off to `odoo-plan`. | `odoo-plan`, `odoo-architecture` |
| `odoo-plan` | task | Sprint task skills | Ordered execution plan with file map, acceptance criteria, test matrix, rollout notes, and open decisions. | Use this skill only when the user wants a plan artifact. If the user wants a risk brief, use `odoo-think`. If the user wants code edits, use `odoo-build`. | `odoo-think`, `odoo-test`, `odoo-upgrade-migration` |
| `odoo-build` | task | Sprint task skills | Concrete in-scope code or configuration changes plus noted assumptions and follow-up validation needs. | Use this skill only when the primary requested output is changed code or config artifacts. If the user is still deciding approach, use `odoo-plan`. | `odoo-orm-modeling`, `odoo-view-ui`, `odoo-security` |
| `odoo-review` | task | Sprint task skills | Structured findings on an existing diff or artifact, separated into required fixes, open risks, and optional improvements. | Use this skill only when the primary requested output is review findings on an existing artifact. If the user wants executed validation evidence, use `odoo-test`. If the prompt centers on exposure or trust boundaries, compose with or defer to `odoo-security`. | `odoo-security`, `odoo-testing-reference` |
| `odoo-test` | task | Sprint task skills | Current-change validation evidence or validation plan tied to a specific diff, addon, bug, or runtime scenario. | Use this skill only when the primary requested output is validation for a concrete change now. If the user asks which Odoo test framework or tags to use in general, hand off to `odoo-testing-reference`. | `odoo-testing-reference`, `odoo-performance` |
| `odoo-ship` | task | Sprint task skills | Go or no-go rollout checklist with staging verification, release sequencing, rollback cautions, and production notes. | Use this skill only when the user wants release readiness or rollout sequencing. If the user needs CLI flag semantics or environment mechanics, hand off to `odoo-delivery-ops`. If the primary issue is data or schema evolution, compose with `odoo-upgrade-migration`. | `odoo-delivery-ops`, `odoo-upgrade-migration`, `odoo-test` |
| `odoo-reflect` | task | Sprint task skills | Retrospective with lessons, remaining gaps, and concrete follow-up actions. | Use this skill only when the primary requested output is a retrospective or lessons learned artifact after work already happened. | `odoo-review`, `odoo-test` |
| `odoo-architecture` | reference | Technical reference skills | Architecture recommendation with target addon, dependency rationale, and bridge-module guidance. | Use this skill only when the primary requested output is an addon-placement or dependency decision. | `odoo-plan`, `odoo-build` |
| `odoo-orm-modeling` | reference | Technical reference skills | ORM modeling guidance with field or method recommendations and anti-pattern checks. | Use this skill only when the primary requested output is a model or ORM design decision. | `odoo-build`, `odoo-performance`, `odoo-security` |
| `odoo-view-ui` | reference | Technical reference skills | UI guidance with view strategy, inheritance notes, and action or menu recommendations. | Use this skill only when the primary requested output is a view, action, menu, template, or client-side UI decision. | `odoo-build`, `odoo-business-website-ecommerce`, `odoo-architecture` |
| `odoo-security` | reference | Technical reference skills | Security assessment with trust boundaries, permission checks, exposure risks, and required safeguards. | Use this skill only when the primary requested output is a trust-boundary or exposure assessment. If the user wants a broad code review, use `odoo-review` and compose with this skill only when security is central. | `odoo-review`, `odoo-build` |
| `odoo-testing-reference` | reference | Technical reference skills | Framework and test-authoring guidance for future or current test design, without claiming current-change readiness. | Use this skill only when the primary requested output is framework or test-authoring guidance. If the user asks whether current work is tested or safe, use `odoo-test`. | `odoo-test`, `odoo-performance`, `odoo-view-ui` |
| `odoo-performance` | reference | Technical reference skills | Performance guidance with hotspot hypotheses, optimization levers, and validation ideas. | Use this skill only when the primary requested output is performance diagnosis or optimization guidance. | `odoo-test`, `odoo-orm-modeling`, `odoo-review` |
| `odoo-integration-api` | reference | Technical reference skills | Integration design guidance with auth model, data flow, safety considerations, and operational constraints. | Use this skill only when the primary requested output is an external integration design or RPC/API decision. | `odoo-security`, `odoo-delivery-ops`, `odoo-build` |
| `odoo-upgrade-migration` | reference | Technical reference skills | Migration strategy with data-preservation plan, script notes, rollout order, and validation needs. | Use this skill only when the primary requested output is migration or upgrade strategy. If the user mainly wants command semantics, use `odoo-delivery-ops`. | `odoo-ship`, `odoo-plan`, `odoo-delivery-ops` |
| `odoo-delivery-ops` | reference | Technical reference skills | Operational mechanics guidance covering commands, flags, environment behavior, and runtime cautions. | Use this skill only when the primary requested output is command, flag, worker, or environment semantics. If the user asks whether a release is ready, use `odoo-ship`. | `odoo-ship`, `odoo-test`, `odoo-upgrade-migration` |
| `odoo-business-sales` | reference | Business reference skills | Sales process map from a backend sales entrypoint, with downstream documents, roles, and cross-app impacts. | Use this skill only when the primary business entrypoint is lead, opportunity, quotation, salesperson action, or backend sales order. If the entrypoint is page, form, cart, checkout, or portal, use `odoo-business-website-ecommerce`. | `odoo-business-inventory`, `odoo-business-accounting`, `odoo-business-website-ecommerce` |
| `odoo-business-purchase` | reference | Business reference skills | Purchase process map with entrypoint, downstream documents, roles, and cross-app impacts. | Use this skill only when the primary business entrypoint is RFQ, purchase order, replenishment through buying, or vendor-side procurement. | `odoo-business-inventory`, `odoo-business-accounting`, `odoo-business-manufacturing` |
| `odoo-business-inventory` | reference | Business reference skills | Inventory process map from a warehouse or stock-document entrypoint, with route logic, valuation, and cross-app effects. | Use this skill only when the primary business entrypoint is receipt, delivery, transfer, route, replenishment, or stock move. If the entrypoint is BoM, MO, work center, routing, or subcontracting, use `odoo-business-manufacturing`. | `odoo-business-sales`, `odoo-business-purchase`, `odoo-business-accounting`, `odoo-business-manufacturing` |
| `odoo-business-manufacturing` | reference | Business reference skills | Manufacturing process map from a BoM or MO entrypoint, with operations, stock effects, and downstream cross-app impacts. | Use this skill only when the primary business entrypoint is BoM, MO, work center, routing, subcontracting, or production planning. If the entrypoint is warehouse documents or routes without production-control logic, use `odoo-business-inventory`. | `odoo-business-inventory`, `odoo-business-purchase`, `odoo-business-sales`, `odoo-business-accounting` |
| `odoo-business-accounting` | reference | Business reference skills | Accounting process map with finance documents, posting impacts, and cross-app implications. | Use this skill only when the primary business entrypoint is invoice, bill, payment, tax, reconciliation, localization, or valuation impact. | `odoo-business-sales`, `odoo-business-purchase`, `odoo-business-inventory`, `odoo-business-expenses` |
| `odoo-business-hr` | reference | Business reference skills | HR process map from a workforce entrypoint, with employee lifecycle touchpoints and cross-app dependencies. | Use this skill only when the primary business entrypoint is employee, contract, attendance, time off, planning, or payroll prerequisite. If the entrypoint is expense claim, expense sheet, reimbursement, or posting, use `odoo-business-expenses`. | `odoo-business-timesheet-project-services`, `odoo-business-expenses`, `odoo-business-accounting` |
| `odoo-business-timesheet-project-services` | reference | Business reference skills | Service-delivery process map with task, time, billing, and cross-app impacts. | Use this skill only when the primary business entrypoint is project, task, timesheet, helpdesk, planning, or service delivery. | `odoo-business-sales`, `odoo-business-hr`, `odoo-business-accounting` |
| `odoo-business-expenses` | reference | Business reference skills | Expense process map from an expense-claim entrypoint, with states, roles, accounting impacts, and related HR links. | Use this skill only when the primary business entrypoint is expense claim, expense sheet, reimbursement, or expense posting. If the entrypoint is employee contract, attendance, time off, or payroll prerequisite, use `odoo-business-hr`. | `odoo-business-hr`, `odoo-business-accounting`, `odoo-security` |
| `odoo-business-website-ecommerce` | reference | Business reference skills | Website and ecommerce process map from a public or portal entrypoint, with customer journey, downstream sales flow, and cross-app impacts. | Use this skill only when the primary entrypoint is page, form, portal, cart, checkout, or online storefront behavior. If the entrypoint is lead, quotation, salesperson action, or backend sales order, use `odoo-business-sales`. | `odoo-business-sales`, `odoo-business-inventory`, `odoo-view-ui`, `odoo-security` |
